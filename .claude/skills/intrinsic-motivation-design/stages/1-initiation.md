# Stage 1: Initiation

Getting someone from "I downloaded this" to "I've done this a few times and it's
becoming part of my routine." First days to ~3 weeks.

## What's happening

The user has intent but no habit. The behaviour lives entirely in the deliberate,
goal-directed system (dorsomedial striatum). Every session requires a conscious
decision: "I should go practise." This is the most fragile phase — the user is one
bad experience or one busy day away from never opening the app again.

Neurologically, early repetitions produce the largest automaticity gains (Lally's
asymptotic curve). The first 10 sessions matter more than the next 50 for habit
formation. Context consistency is critical: same time, same situation, same cue.

External scaffolding is appropriate and necessary here. The goal isn't to build
intrinsic motivation yet — it's to get enough repetitions that the behaviour starts
transferring to automatic systems. Intrinsic motivation emerges from competence, and
competence requires practice. You can't feel competent at something you've done twice.

## Good patterns for this stage

### Make the first action absurdly small

**Why:** Fogg's B=MAP model — behaviour happens when motivation x ability x prompt
all cross the threshold simultaneously. You can't reliably control motivation, but
you can make ability near-infinite by making the action tiny. A 30-second task
attached to an existing routine will happen far more reliably than a 15-minute
session requiring willpower.

**Do this:**

- Default first session to something completable in under 2 minutes
- Frame it as "just try one" not "complete your first lesson"
- Let the user naturally expand from the tiny start (most will — starting is the hard
  part, not continuing)
- Never show the user how much more there is to do. The full curriculum is
  overwhelming at this stage.

**Don't do this:**

- Mandatory onboarding tutorials longer than 60 seconds
- "Set your daily goal!" prompts before the user has even tried the activity
- Showing progress bars that highlight how little they've done (0.5% complete!)

### Anchor to existing routines

**Why:** The basal ganglia learns cue → routine → reward sequences. Attaching a new
behaviour to an existing cue (something the user already does reliably) borrows the
existing neural pathway. "After I pour my morning coffee" is a cue that fires
automatically; "at some point during the day" requires executive function every time.

**Do this:**

- During onboarding (not before the first session — let them try it first), suggest
  anchor points: "When do you want to practise? After morning coffee / During lunch
  / Before bed"
- Frame notifications around the chosen anchor, not arbitrary times
- If the user doesn't pick an anchor, default to a consistent daily time and let them
  change it

**Don't do this:**

- Multiple notifications at different times ("Haven't practised yet today!")
- Notifications that feel like nagging rather than gentle cues
- Asking the user to commit to a specific duration before they know what a session
  feels like

### Use external prompts (that plan their own obsolescence)

**Why:** At this stage, the user genuinely needs reminding. The behaviour isn't
automatic yet. But notifications that persist forever become controlling — the user
starts practising to silence the notification rather than because they want to. The
research shows this is the path to resentment and uninstalls.

**Do this:**

- Daily reminders for weeks 1-2
- Reduce to every-other-day by week 3
- Include a "You seem to have this — want fewer reminders?" prompt around day 14
- Make the notification content informational, not controlling: "Your Greek is waiting"
  not "Don't break your streak!"
- Always include a one-tap "Not today" that doesn't trigger guilt

**Don't do this:**

- Escalating notification urgency ("You haven't practised in 2 days!")
- Loss-framed notifications at any stage
- Notifications that require multiple taps to dismiss
- Re-engagement campaigns that feel desperate

### Show the activity, not the system

**Why:** New users need to understand what they'll actually be doing, not how the
app's reward system works. Onboarding that explains XP, streaks, leaderboards, and
badge systems before the user has done anything is teaching them to care about the
wrong things. It front-loads external locus.

**Do this:**

- First screen after download: the core activity itself. A drill, a flashcard, a
  practice problem. Let them do the thing.
- Explain app mechanics only when they become relevant (show the review schedule
  when a card comes up for review, not during onboarding)
- If you have a demo/preview, make it interactive — let the user experience the
  feeling of doing the activity, not just watch a video about it

**Don't do this:**

- Onboarding carousels explaining the reward system
- "Here's how XP works!" before the first session
- Mandatory account creation before the first interaction with the core activity

### Allow early wins — but honest ones

**Why:** Competence needs to start building immediately, but false competence is
detectable and counterproductive. The user should feel "I can do this" after session
1, but because they actually did something real, not because the app inflated easy
tasks with "Amazing!" praise.

**Do this:**

- Start with genuinely achievable content that still teaches something real
- Let the user see what they produced: the forms they conjugated, the words they
  learned, the problem they solved
- Acknowledge the completion descriptively: "You learned 5 new verb forms"
- If the app has a preview of real-world application, show it early: "You can now
  say [useful phrase]"

**Don't do this:**

- Celebration screens after trivially easy first tasks (this is where the Duolingo
  confetti is most damaging — it trains the user from minute one that the app's
  reaction is the payoff, not the learning)
- Difficulty so low it feels patronising
- "You're a natural!" after task 1 (person-praise, fixed mindset, also obviously
  false — they did one thing)

## Bad patterns for this stage

### Front-loading the reward system explanation

**Why it's tempting:** You want the user to understand the value proposition.
**Why it fails:** The reward system IS the app in the user's mind from that point on.
You've told them: the reason to use this app is streaks, XP, and leaderboards. Now
every session is in service of those mechanics rather than the underlying goal.

### Asking for commitment before experience

**Why it's tempting:** Commitment devices work (StickK data: 3x success rate with
stakes).
**Why it fails at this stage:** The user doesn't know what they're committing to.
A "Set your daily goal: 5 min / 10 min / 15 min" prompt before the first session
is meaningless — they have no reference for what 10 minutes of this activity feels
like. Worse, if they set 15 minutes and find sessions exhausting at 8 minutes,
they've failed on day 1 by a standard they set without information. Let them
experience it first. Goal-setting belongs in Stage 2, after they have a baseline.

### Heavy celebration on session 1

**Why it's tempting:** You want the user to feel good and come back.
**Why it fails:** It trains the dopamine system to expect the celebration. By session
5, the same celebration produces no prediction error (fully predicted = zero
dopamine signal). By session 10, the user is noticing the celebration is the same
every time and it feels hollow. You've habituated your strongest engagement tool
before the habit has formed. Save the emotional weight for moments that genuinely
merit it — or better, don't use it at all and let the activity itself be the reward.

## The completion moment at Stage 1

At this stage, the user doesn't have enough data for meaningful deltas and hasn't
built enough competence for struggle-reframes to land. Keep completion moments
extremely light:

1. **What happened:** "You practised 8 present-tense forms. 6 minutes."
2. **An anchor prompt** (first few sessions only): "Want to practise at the same time
   tomorrow? Habits form faster with consistent timing."
3. **The fork:** "Try 5 more / Explore a different tense / Done for today"

That's it. No celebration. No evaluation. No score. The user just did something new,
and the app noticed. The emotional response is theirs to have internally.
